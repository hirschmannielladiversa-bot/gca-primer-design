1. What this package does

Enter one species name and the following is executed automatically:

1. NCBI sequence fetch → 2. Primer3 design → 3. Crosscheck → 4. Report generation → 5. DNA Dynamo .cow visualization

The main use case is primer design for GCA (Gut Content Analysis). You want primers that amplify the target species' DNA but NOT non-target or related species.

Strengths of this package

  • Enter one species name — the rest is automatic
  • Automatic fallback for Lepidoptera AT-rich mtDNA
  • Target / non-target / all sequences / excluded species each visualized as separate DNA Dynamo .cow files
  • Full DNA Dynamo automation on macOS (via the Quartz API)

2. Installation

Requirements

RequirementNotes
macOS10.15 or later recommended. DNA Dynamo automation is macOS-only
Python 3.10+Homebrew or python.org build
DNA DynamoOnly needed for the visualization step. Download from the official site

Setup

# 1. Clone the repository
git clone https://github.com/hirschmannielladiversa-bot/gca-primer-design.git
cd gca-primer-design

# 2. Install Python dependencies
pip3 install -r requirements.txt

# 3. (Optional) For DNA Dynamo automation
pip3 install pyobjc-framework-Quartz

# 4. Make shell scripts executable
chmod +x auto_full.sh auto_visualize.sh verify_slw_count.sh

NCBI Entrez API email

NCBI's terms of service require a valid email address in API requests. Set the environment variable GCA_ENTREZ_EMAIL to your own email:

export GCA_ENTREZ_EMAIL="you@your-institution.edu"

macOS accessibility permission

When using DNA Dynamo automation, macOS will ask "Allow Terminal to control other applications?" Grant permission, or add Terminal (and python3) under System Settings → Privacy & Security → Accessibility.

3. Quick Start (3 minute walkthrough)

Shortest path for first-time users:

# 1. cd to the package
cd ~/gca-primer-design

# 2. Just run
./auto_full.sh

Interactive prompts:

Enter target species (e.g. Helicoverpa armigera): Helicoverpa armigera
Enter target gene (e.g. COI, matK): COI
Related species (comma-separated) [optional]: Helicoverpa assulta,Spodoptera exigua
Non-target organisms (comma-separated) [optional]: Spodoptera litura
Use AT-rich mode? (Lepidoptera etc. mtDNA) [y/N]: y

【Confirmation】
  Target       : Helicoverpa armigera
  Gene         : COI
  Related      : Helicoverpa assulta,Spodoptera exigua
  Non-target   : Spodoptera litura
  AT-rich      : ON

Press Enter to start (Ctrl+C to cancel)...

Pressing Enter runs automatically:

  1. Fetch sequences from NCBI
  2. Design primer candidates with Primer3
  3. Run crosscheck against related / non-target species
  4. Generate report
  5. Generate 4 .cow files using the top-recommended primer
  6. Launch DNA Dynamo and auto-align each category in turn

When finished:

reports/Helicoverpa_armigera_COI/プライマー結合/
├── ターゲット配列のみ.cow         (Target sequences only)
├── 非標的配列のみ.cow             (Non-target only)
├── すべての配列.cow               (All sequences)
└── 除外した種_Spodoptera_litura.cow (Excluded species)

Important: Do NOT touch the screen while DNA Dynamo automation is running. Moving the mouse or pressing keys will throw AppleScript interactions to the wrong place and the run will fail.

4. Terminal usage — detailed

4.1 Opening a terminal

On macOS, press ⌘ + Space to open Spotlight, type Terminal, and press Return.

4.2 Navigating to the package

Use cd to go to where you cloned the package. If you cloned under your home directory:

cd ~/gca-primer-design

If the path contains spaces, always quote it:

cd "/Volumes/External SSD/gca-primer-design"

Warning about paths with spaces

zsh wraps long lines visually in the terminal, so pasting a multi-line command may split at the wrong place and break arguments. Either keep the command on a single line, or save it into a shell script.

4.3 Running scripts

There are three main entry points:

ScriptPurpose
./auto_full.shSpecies name → DNA Dynamo .cow files (recommended)
./auto_visualize.sh <PROJECT>Re-visualize an already-designed project
./verify_slw_count.sh <PROJECT>Verify SLW counts of generated .cow files

4.4 Interactive mode

Run without arguments to start the 5-question prompt:

./auto_full.sh
PromptExampleRequired?
Target speciesHelicoverpa armigerarequired
Target geneCOI / matK / rbcLrequired
Related species (comma-separated)Helicoverpa assulta,Spodoptera exiguaoptional
Non-target organisms (comma-separated)Spodoptera lituraoptional
AT-rich modey or noptional (default n)

Related / non-target / excluded — what's the difference?

  • Related species (related): phylogenetically close to the target. Primer3 uses them as negative reference — "the primer should NOT bind here"
  • Non-target organisms (nontarget): other species that may co-occur in the analyzed sample. Same role as related
  • Excluded species (excluded): sequences removed by quality filtering (misidentified records, low similarity to others, etc.)

4.5 CLI argument mode

Pass everything as flags to skip the prompts. Useful for scripting / batching:

# Target + gene only (related/nontarget empty, AT-rich auto)
./auto_full.sh "Helicoverpa armigera" COI

# Explicit AT-rich
./auto_full.sh "Helicoverpa armigera" COI --at-rich

# Related + non-target
./auto_full.sh "Bemisia tabaci" COI \
  --related "Trialeurodes vaporariorum,Aleyrodes proletella" \
  --nontarget "Aphis gossypii"

# Already designed — just re-run visualization
./auto_full.sh "Helicoverpa armigera" COI --skip-design

# Don't launch DNA Dynamo
./auto_full.sh "Helicoverpa armigera" COI --no-automation

5. What is AT-rich mode?

Many insect mitochondrial genomes — especially Lepidoptera (moths/butterflies), Diptera (flies), and Hymenoptera (bees/wasps) — have AT content above 70%. For example:

TaxonmtDNA AT content
Helicoverpa armigera (cotton bollworm)~80%
Bombyx mori (silkworm)~81%
Drosophila melanogaster~78%
Mammalian mtDNA (reference)~55%

Why is this a problem?

Primer3's default parameters assume "GC content 40–60%, Tm 57–63°C". For AT-rich templates, primers satisfying these constraints simply don't exist, so the design step returns zero candidates.

How this package handles it

When --at-rich is enabled:

ParameterDefault--at-rich
PRIMER_MIN_TM57°C50°C
PRIMER_MAX_TM63°C60°C
PRIMER_MIN_GC40%20%
PRIMER_MAX_GC60%60%

No decision required — automatic fallback

If Primer3 returns "no candidates found" with default settings, auto_full.py automatically retries with --at-rich. Even if you're not sure whether your species is AT-rich, just run with defaults first.

7. About DNA Dynamo automation

DNA Dynamo is written in Java Swing. On macOS, the standard accessibility API (System Events) does not expose menu items for Java apps. This package works around that with its own technique:

  1. Open the cow file
  2. Find the Align To button on the toolbar by walking the AX tree and click it
  3. When the Drag Drop Assembly Window appears, locate the target FASTA's static text by Y coordinate
  4. Compute the rightmost pixel of its row and click via the Quartz API (the real checkbox position)
  5. Click Assemble Sequences → Align Selected Sequences → auto-save with Cmd+S

During execution

  • Do not move the mouse
  • Do not press any keys
  • Do not switch to another app
  • Do not click notifications

If something fails, fully quit DNA Dynamo with ⌘+Q and try again.

8. Troubleshooting

"No primer candidates found"
Likely AT-rich sequences. auto_full.sh should automatically retry with --at-rich. If that still fails, try increasing --max-seqs 20 to pull more sequences.
Cannot fetch sequences from NCBI
  • Check your internet connection
  • Set GCA_ENTREZ_EMAIL to your own email
  • Double-check the species spelling (e.g. Helicoverpa armigera — capitalized genus, single space)
  • NCBI may have temporary issues. Wait a few minutes and retry
DNA Dynamo automation fails
  1. System Settings → Privacy & Security → Accessibility: make sure Terminal and python3 are checked
  2. Fully quit DNA Dynamo with ⌘+Q and re-run
  3. Absolutely do not touch the screen during execution
  4. Use --no-automation to just generate files and open them manually later
"Quartz not found" error 
pip3 install pyobjc-framework-Quartz
For Homebrew Python: /opt/homebrew/bin/pip3 install pyobjc-framework-Quartz
Paths with spaces cause errors
Wrap in double quotes: cd "/Volumes/tkg SSD/..."
If zsh line-continuation (backslash+newline) splits your command, save it to a shell script and execute that instead.
cow file opens in Linear Editor instead of alignment view
Run with DNA Dynamo automation enabled (do not pass --no-automation). Verify with ./verify_slw_count.sh <PROJECT> — SLW count ≥ 2 means the alignment was embedded successfully.

9. Q&A

Does this work on Windows?
Sequence fetch, primer design, and cow-file generation all work on Windows. However, DNA Dynamo automation is macOS-only (it uses AppleScript and Quartz). On Windows, use --no-automation to generate files and open them manually in DNA Dynamo.
Does this work on Linux?
Sequence fetch, design, and report generation work on Linux. DNA Dynamo itself runs only on macOS/Windows, so the cow automation step is not available on Linux.
How long does one species take?
Depends on your network, but roughly:
  • NCBI fetch: 30 seconds – 1 minute
  • Primer3 design: 5 seconds
  • Crosscheck: 5 seconds
  • Report generation: 1 second
  • DNA Dynamo automation (4 categories): 1 – 3 minutes
Roughly 3 – 5 minutes total.
How is the "most recommended" primer chosen?
The row with the highest Quality_Score in primer_candidates.csv is selected. This score combines Primer3's score with target specificity. Use auto_visualize.py --top-n 3 to process the top 3 candidates (they'll be saved in subfolders).
How do I run another species?
Just run ./auto_full.sh again and enter a different species name. Each species is isolated in its own reports/<Species>_<Gene>/ folder, so multiple species can coexist.
What happens if I don't specify related species?
Design still runs, but there's no cross-reactivity evaluation. Whether the resulting primer is actually target-specific must then be verified by other means. Specifying related species is strongly recommended.
Is my personal info sent to NCBI via the Entrez email?
NCBI's terms of service require a valid email address in API requests to track potential abuse. In normal usage NCBI will not contact you. Any address you own is fine; a throwaway alias works too.
How do I reuse fetched sequences offline?
FASTA files under reports/<Species>_<Gene>/sequences/ can be reused directly. Pass --skip-design next time to skip NCBI fetch and reuse existing files.
How much can I trust in-silico primer design before lab testing?
This package runs Primer3 design and mismatch-based crosscheck, but you must validate empirically (PCR, gel, sequencing). In-silico design alone cannot fully predict:
  • Effects of secondary structure
  • Binding to unknown related species not in NCBI
  • Effects of template quality
Can I visualize a custom primer?
Use scripts/generate_primer_cow.py directly with manual Forward/Reverse sequences (no CSV required): 
python3 scripts/generate_primer_cow.py \
  --input-fasta path/to/your.fasta \
  --input-label TARGET \
  --forward CTTATCTTTTGATCAAATGCCA \
  --reverse ATGTATTAAGGTTTCGATCTGT \
  --output output.cow \
  --primer-name MyPrimer
How do I verify generated cow files? 
./verify_slw_count.sh Helicoverpa_armigera_COI
SLW count of 2 or more (consensus + 1+ sequences) indicates success. Target-only: 11 (consensus + 10); all: 25 (consensus + 24).
How do I report bugs or request features?
Open an issue on GitHub. Please include reproduction steps, the exact error message, your OS version, and Python version.
