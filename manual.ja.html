1. このパッケージは何をするのか

標的種の名前を 1 つ入力するだけで、以下のすべてを自動実行します:

1. NCBI 配列取得 → 2. Primer3 設計 → 3. クロスチェック → 4. レポート生成 → 5. DNA Dynamo .cow 可視化

主な用途は GCA (Gut Content Analysis: 消化管内容物 DNA 解析) 用のプライマー設計です。標的種の DNA を増幅し、近縁種・非標的種・除外種に対しては増幅しないプライマーを設計します。

このパッケージの強み

  • 種名を 1 行入れるだけで完結
  • Lepidoptera (鱗翅目) の AT-rich 配列にも自動対応
  • 標的・非標的・全配列・除外種それぞれを別の .cow ファイルとして可視化
  • DNA Dynamo まで完全自動操作 (macOS の Quartz API 経由)

2. インストール

前提環境

必須説明
macOS10.15 以降推奨。DNA Dynamo 自動操作部分は macOS 専用
Python 3.10 以降Homebrew 版または python.org 版
DNA Dynamo可視化を使う場合のみ必須 (公式サイトから入手)

セットアップ手順

# 1. パッケージを取得
git clone https://github.com/TKG-M/gca-primer-design.git
cd gca-primer-design

# 2. 依存パッケージをインストール
pip3 install -r requirements.txt

# 3. (DNA Dynamo 自動操作を使う場合のみ) Quartz をインストール
pip3 install pyobjc-framework-Quartz

# 4. シェルスクリプトに実行権限を付与
chmod +x auto_full.sh auto_visualize.sh verify_slw_count.sh

NCBI Entrez API の設定

scripts/fetch_sequences.py 内の Entrez.email を あなた自身のメールアドレスに書き換えてください。これは NCBI の利用規約で必須です (無視するとアクセスがブロックされる可能性があります)。

macOS のアクセシビリティ許可

DNA Dynamo 自動操作を使う場合、初回実行時に macOS が「Terminal が他のアプリを操作することを許可しますか？」と聞いてきます。
許可するか、システム設定 → プライバシーとセキュリティ → アクセシビリティ で Terminal (および python3) にチェックを入れてください。

3. クイックスタート (3 分で体験)

初めて使う場合の最短手順:

# 1. ターミナルでパッケージのフォルダへ移動
cd ~/gca-primer-design

# 2. これだけ実行
./auto_full.sh

すると以下のように対話入力が始まります:

標的種を入力してください (例: Helicoverpa armigera): Helicoverpa armigera
対象遺伝子を入力してください (例: COI, matK): COI
近縁種をカンマ区切りで入力 [空欄可]: Helicoverpa assulta,Spodoptera exigua
非標的生物をカンマ区切りで入力 [空欄可]: Spodoptera litura
AT-rich モードを使用しますか？ (Lepidoptera 等の mtDNA) [y/N]: y

【入力確認】
  標的種       : Helicoverpa armigera
  遺伝子       : COI
  近縁種       : Helicoverpa assulta,Spodoptera exigua
  非標的生物   : Spodoptera litura
  AT-rich      : ON

Enter で開始 (Ctrl+C で中止)...

Enter を押すと、自動で:

  1. NCBI から配列を取得
  2. Primer3 でプライマー候補を設計
  3. クロスチェック
  4. レポート生成
  5. 最推奨プライマーで 4 種類の .cow ファイルを生成
  6. DNA Dynamo を起動して順次アラインメント表示・保存

完了後、以下が出力されます:

reports/Helicoverpa_armigera_COI/プライマー結合/
├── ターゲット配列のみ.cow
├── 非標的配列のみ.cow
├── すべての配列.cow
└── 除外した種_Spodoptera_litura.cow

重要: DNA Dynamo 自動操作中は 絶対に画面に触れないでください。マウスを動かしたりキーを押したりすると、AppleScript の操作が誤った場所に飛んで失敗します。

4. ターミナルでの使い方 — 詳しい解説

4.1 ターミナルを開く

macOS で ⌘ + Space を押して Spotlight を開き、Terminal と入力して Return。

4.2 作業ディレクトリへ移動

パッケージを置いた場所へ cd で移動します。例えばホームディレクトリの直下に置いたなら:

cd ~/gca-primer-design

外部ドライブに置いた場合 (パスにスペースが含まれる場合は必ずダブルクォートで囲む):

cd "/Volumes/tkg SSD/gca_primer_optimization/gca_auto_design_package"

パスにスペースがある場合の注意

zsh は引数の長い行を画面で折り返し表示するため、コピペすると改行が混入することがあります。1 行で書くか、シェルスクリプトに保存してから実行してください。

4.3 スクリプトの実行

3 つの主要なエントリポイントがあります:

スクリプト用途
./auto_full.sh種名→DNA Dynamo まで完全自動 (推奨)
./auto_visualize.sh <PROJECT>既に設計済みのプロジェクトの可視化のみ
./verify_slw_count.sh <PROJECT>生成された .cow ファイルの SLW 数を検証

4.4 対話入力モード

引数なしで実行すると 5 つの質問が順次出ます:

./auto_full.sh
質問入力例必須
標的種Helicoverpa armigera必須
対象遺伝子COI / matK / rbcL必須
近縁種 (カンマ区切り)Helicoverpa assulta,Spodoptera exigua任意
非標的生物 (カンマ区切り)Spodoptera litura任意
AT-rich モードy または n任意 (デフォルト n)

近縁種・非標的種・除外種の違い

  • 近縁種 (related): 標的と近い系統の種。プライマー設計時に「ここには結合してほしくない」配列として参照
  • 非標的 (nontarget): 解析サンプル中に含まれる可能性がある別種。同上
  • 除外 (excluded): 配列フィルタリングで除外された配列 (品質や種同定不一致など)

4.5 CLI 引数モード

引数で全部指定すれば対話なしで一発実行できます。スクリプトやバッチ処理に便利:

# 標的種と遺伝子だけ指定 (近縁種・非標的なし、AT-rich は自動判定)
./auto_full.sh "Helicoverpa armigera" COI

# AT-rich を明示的に有効化
./auto_full.sh "Helicoverpa armigera" COI --at-rich

# 近縁種と非標的を指定
./auto_full.sh "Bemisia tabaci" COI \
  --related "Trialeurodes vaporariorum,Aleyrodes proletella" \
  --nontarget "Aphis gossypii"

# 既に設計済みなら --skip-design で時間短縮
./auto_full.sh "Helicoverpa armigera" COI --skip-design

# DNA Dynamo を起動せずファイル生成のみ
./auto_full.sh "Helicoverpa armigera" COI --no-automation

引数一覧

引数説明デフォルト
--target / -t標的種 (例: "Helicoverpa armigera")対話入力
--gene / -g対象遺伝子 (例: COI)対話入力
--related近縁種 (カンマ区切り)空
--nontarget非標的生物 (カンマ区切り)空
--at-richAT-rich モードを有効化OFF (失敗時は自動 ON)
--max-seqs種ごとの最大取得配列数10
--skip-designfetch〜report をスキップ (可視化のみ)OFF
--no-automationcow 生成のみで DNA Dynamo は起動しないOFF
--categories可視化するカテゴリ (target/nontarget/all/excluded)全部
--non-interactive対話入力を完全にスキップOFF

5. AT-rich モードとは何か

多くの昆虫 (特に Lepidoptera 鱗翅目, Diptera 双翅目, Hymenoptera 膜翅目) のミトコンドリア DNA は、AT 含有率が 70% を超えることが知られています。例:

分類群mtDNA AT 含有率
Helicoverpa armigera (オオタバコガ)~80%
Bombyx mori (カイコ)~81%
Drosophila melanogaster~78%
哺乳類 mtDNA (参考)~55%

何が問題か

Primer3 のデフォルトパラメータは「GC 含有率 40–60%、Tm 57–63°C」を前提にしています。AT-rich 配列ではこの条件を満たすプライマーが そもそも存在しないため、設計が失敗します。

このパッケージの対応

--at-rich を有効化すると以下が緩和されます:

パラメータデフォルト--at-rich
PRIMER_MIN_TM57°C50°C
PRIMER_MAX_TM63°C60°C
PRIMER_MIN_GC40%20%
PRIMER_MAX_GC60%60%

判断不要 — 自動フォールバック

auto_full.py は Primer3 が「候補が見つかりませんでした」と返した場合、自動的に --at-rich で再試行します。AT-rich かどうか分からなくても、まずデフォルトで実行すれば大丈夫です。

6. 出力されるもの

1 回の ./auto_full.sh 実行で、以下のフォルダ構造が作られます:

reports/Helicoverpa_armigera_COI/
├── sequences/                          ← STEP 1 (NCBI 取得)
│   ├── target_sequence.fasta
│   ├── related_sequences.fasta
│   ├── all_COI_sequences.fasta
│   └── excluded_*.fasta
├── primer_candidates/                  ← STEP 2 (Primer3)
│   ├── primer_candidates.csv
│   ├── primer_candidates.json
│   └── primer_candidates.txt
├── crosscheck_results/                 ← STEP 3 (クロスチェック)
│   └── *.tsv
├── report.md                           ← STEP 4 (総合レポート)
└── プライマー結合/                     ← STEP 5 (DNA Dynamo cow)
    ├── ターゲット配列のみ.cow
    ├── 非標的配列のみ.cow
    ├── すべての配列.cow
    └── 除外した種_Spodoptera_litura.cow

各 .cow ファイルの中身

ファイル名内容確認できること
ターゲット配列のみ.cow標的種の全配列 + プライマー結合位置標的種に対する結合の安定性
非標的配列のみ.cow近縁種の全配列 + プライマー結合位置クロスリアクティビティのリスク
すべての配列.cow標的+近縁+全部 + プライマー結合位置標的と非標的のミスマッチ差
除外した種_*.cow除外された配列 + プライマー結合位置除外判断の妥当性検証

7. DNA Dynamo 自動操作について

DNA Dynamo は Java Swing で書かれているため、macOS の標準的なアクセシビリティ API (System Events) では メニュー項目が公開されません。このパッケージは独自の方法でこの制限を回避しています:

  1. cow ファイルを開く
  2. cow ウィンドウのツールバーから Align To ボタンを AX tree から探してクリック
  3. Drag Drop Assembly Window が出たら、対象 FASTA の static text を Y 座標で特定
  4. そのチェックボックスの 右端ピクセル座標を計算して Quartz API で物理クリック
  5. Assemble Sequences → Align Selected Sequences → 自動保存 (Cmd+S)

実行中の注意

  • マウスを動かさない
  • キーボードに触らない
  • 他のアプリに切り替えない
  • 通知をクリックしない

失敗した場合は DNA Dynamo を ⌘+Q で完全終了し、再実行してください。

8. トラブルシューティング

「プライマー候補が見つかりませんでした」と言われた
AT-rich 配列の可能性が高いです。auto_full.sh なら自動で --at-rich 再試行されます。それでも失敗する場合は --max-seqs 20 など取得数を増やしてください。
NCBI から配列が取得できない
  • インターネット接続を確認
  • scripts/fetch_sequences.py の Entrez.email が自分のメールに設定されているか確認
  • 種名のスペルミスがないか確認 (例: Helicoverpa armigera ← 大文字始まり、スペース 1 つ)
  • NCBI 側の障害の可能性。数分待って再実行
DNA Dynamo の自動操作が失敗する
  1. システム設定 → プライバシーとセキュリティ → アクセシビリティ で Terminal と python3 にチェック
  2. DNA Dynamo を ⌘+Q で完全終了してから再実行
  3. 実行中は画面に絶対触れない
  4. --no-automation を付けてファイル生成のみ実行し、後で手動で開く
Quartz が見つからないエラー 
pip3 install pyobjc-framework-Quartz
Homebrew Python の場合: /opt/homebrew/bin/pip3 install pyobjc-framework-Quartz
パスにスペースが入っているとエラーになる
ダブルクォートで囲む: cd "/Volumes/tkg SSD/..."
zsh の継続行 (バックスラッシュ改行) でコマンドが分断される場合は、シェルスクリプトに保存してから実行してください。
cow ファイルを DNA Dynamo で開いても Linear Editor になってしまう
DNA Dynamo の自動操作 (--no-automation なし) を実行すれば Sequencing Project (アラインメント) として開きます。./verify_slw_count.sh <PROJECT> で SLW 数が 1 以上なら成功しています。

9. Q&A

このパッケージは Windows でも動きますか？
配列取得・プライマー設計・cow ファイル生成までは Windows でも動くはずですが、DNA Dynamo 自動操作の部分は macOS 専用です (AppleScript と Quartz を使うため)。Windows では --no-automation を付けて実行し、生成された cow ファイルを手動で DNA Dynamo で開いてください。
Linux でも動きますか？
配列取得・設計・レポート生成は動きます。DNA Dynamo は macOS / Windows のみのため、Linux では cow 自動操作はサポートされません。
1 つの種でどれくらい時間がかかりますか？
ネットワーク速度にもよりますが、おおよそ:
  • NCBI 取得: 30 秒〜1 分
  • Primer3 設計: 5 秒
  • クロスチェック: 5 秒
  • レポート生成: 1 秒
  • DNA Dynamo 自動操作 (4 カテゴリ): 1〜3 分
合計でおよそ 3〜5 分です。
「最推奨」プライマーはどう決まるんですか？
primer_candidates.csv の Quality_Score 列が最も高いものを使います。これは Primer3 のスコアと標的特異性をベースに計算されています。複数のプライマーを試したい場合は auto_visualize.py --top-n 3 のように指定すると上位 3 件を生成します (サブフォルダ化されます)。
他の種で試したい場合は？
./auto_full.sh をもう一度実行して別の種名を入力するだけです。各種は reports/<Species>_<Gene>/ の独立フォルダに格納されるので、複数の種を並行管理できます。
近縁種を指定しないと何が起きますか？
近縁種なしでも設計は走ります。ただし、その場合「クロスリアクティビティの評価」ができないため、得られたプライマーが本当に標的特異的かは別途確認が必要です。近縁種を指定することを強く推奨します。
Entrez メールアドレスは何のため？個人情報が NCBI に送られる？
NCBI Entrez API の利用規約で、リクエストに有効なメールアドレスを含めることが求められています。これは API の濫用を追跡するためで、通常の使用では NCBI からあなたに連絡が来ることはありません。捨てメールアドレスでも構いません。
取得した配列をオフラインで再利用したい
reports/<Species>_<Gene>/sequences/ 内の FASTA ファイルがそのまま再利用できます。次回実行時に --skip-design を付ければ NCBI 取得をスキップして既存ファイルから処理を再開します。
プライマーを実験で検証する前に、in-silico でどこまで信頼できる？
このパッケージは Primer3 設計 + 近縁種ミスマッチ評価までを行いますが、必ず実験的な検証 (PCR, 電気泳動, シーケンシング) を行ってください。in-silico 設計だけでは:
  • 2 次構造の影響
  • 未知の近縁種への結合
  • テンプレート品質の影響
を完全に予測することはできません。
カスタムプライマーの結合を可視化したい
scripts/generate_primer_cow.py を直接呼ぶことで、CSV を経由せず手動で Forward/Reverse 配列を指定できます: 
python3 scripts/generate_primer_cow.py \
  --input-fasta path/to/your.fasta \
  --input-label TARGET \
  --forward CTTATCTTTTGATCAAATGCCA \
  --reverse ATGTATTAAGGTTTCGATCTGT \
  --output output.cow \
  --primer-name MyPrimer
生成された cow ファイルが正しいか検証したい 
./verify_slw_count.sh Helicoverpa_armigera_COI
SLW count が 2 以上 (consensus + 1 以上の配列) なら成功しています。ターゲット配列のみなら 11 (consensus + 10 配列)、すべての配列なら 25 (consensus + 24 配列) のような値になります。
このパッケージにバグや改善要望がある場合は？
GitHub の Issues に投稿してください。具体的な再現手順、エラーメッセージ、OS バージョン、Python バージョンを含めるとスムーズです。
